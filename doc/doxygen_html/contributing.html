Basics

Commits on the master are not allowed. Create a branch an merge it to the master instead. Merging with master is only allowed if the CI pipeline successes.

Commits

To generate data for the change log use git trailers in your commit message.

Title of the change
Description of the change.
Changelog: category

category should be one of the following

  • added New feature
  • fixed Bug fix
  • changed Feature change
  • deprecated New deprecation
  • removed Feature removal
  • security Security fix
  • performance Performance improvement
  • other Other

Release

The CI pipeline will automatically create a release for a new tag. Make sure to update the changelog before you create a new tag. The changelog can be updated with the following command

curl --header "PRIVATE-TOKEN: TOKEN" "https://gitlab.vi.vector.int/api/v4/projects/305/repository/changelog?version=X.Y.Z"

Replace TOKEN with a token, which you can generate in the preference tab in the gitlab web interface. Replace the X.Y.Z with the tag you will use.
